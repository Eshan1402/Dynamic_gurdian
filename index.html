System Overview

This section introduces the Dynamic Guardian, an AI-powered safety system for vehicles. It operates on a two-board architecture to gather sensor data, process it using machine learning, and provide real-time alerts on a dashboard and to the cloud.

The system combines an **Arduino** for reliable sensor collection and a powerful **ESP32-S3** for ML inference, visualization, and cloud communication. It is designed to detect four key indicators of driver and vehicle state:

Driver Fatigue

Monitors sensor data for signs of drowsiness or low alertness in the driver.

Driver Stress

Detects high heart rate or other erratic biometric behaviors indicating stress.

Pothole / Road Condition

Identifies rough terrain or road vibrations from accelerometer data.

Harsh or Late Braking

Flags rapid deceleration or sudden stops as potential risk events.

How It Works

  1. Data Acquisition (Arduino): All sensors (heart rate, accelerometer, etc.) are read by the dedicated Arduino board.
  2. Transmission (UART): The Arduino formats and sends this data as a clean string to the ESP32.
  3. Reception & Preprocessing (ESP32): The ESP32 receives the string, parses it, and averages the data over a 5-second window.
  4. Inference (ESP32): An Edge Impulse model predicts driver and vehicle states from this averaged data.
  5. Visualization (ESP32): The onboard LVGL dashboard shows real-time fatigue/stress levels and road events.
  6. Cloud Update (ESP32): The latest data and alerts are sent to Firebase Realtime Database for remote analysis.

Configuration & Setup

Setting up the Dynamic Guardian involves flashing code to both the Arduino and the ESP32. Follow these steps to configure your device.

1. Arduino Transmitter

  1. Connect all your sensors (MPU6050, Heart Rate, CO2, etc.) to the Arduino.
  2. In your `Arduino_Transmitter.ino` code, implement the logic to read from each sensor.
  3. Combine all readings into a single comma-separated string and send it over the `Serial` port.
// Example Arduino code snippet
Serial.println(String(hr) + "," + String(hrv) + "," + String(co2) + "," + String(accelZ));

2. ESP32-S3 Receiver

  1. Connect the Arduino's TX pin to the ESP32's RX pin (use a logic level shifter if needed). Connect their GND pins.
  2. In the ESP32 code, update your network credentials:
#define WIFI_SSID "YOUR_WIFI_SSID"
#define WIFI_PASSWORD "YOUR_WIFI_PASSWORD"
  1. Update your Firebase credentials:
#define API_KEY "YOUR_FIREBASE_API_KEY"
#define DATABASE_URL "https://your-project-id.firebaseio.com/"
  1. Ensure your Firebase Realtime Database rules allow writes.
  2. Upload the complete code to the ESP32-S3.
